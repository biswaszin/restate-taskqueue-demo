# Restate Task Queue Demo (Priority + Tracker + Mock Pipeline)

A simple demo pipeline that:
1. Accepts file uploads as tasks
2. Queues tasks by tier-based priority
3. Processes tasks through mock **convert** and **AI** steps
4. Tracks status transitions end-to-end

---

## What this demo shows

- Priority scheduling (`tier1 > tier2 > tier3 > free`)
- FIFO ordering within the same tier
- Task lifecycle tracking:
  - `queued`
  - `processing:convert`
  - `processing:ai`
  - `completed` / `failed`
- Output artifacts:
  - JSON + XML generated by convert mock
  - analysis output generated by AI mock

> Current implementation is in-memory for queue/tracker (demo-friendly), with files written to local `storage/`.

## 1) Setup

From project root:

```bash
cd ~/task-queue/restate-taskqueue-demo
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

---

## 2) Package/import sanity (important)

Make sure these files exist (fixes import issues in tests):

```bash
touch app/__init__.py
touch restate_services/__init__.py
touch mocks/__init__.py
touch tests/__init__.py
```

Create `pytest.ini` in project root:

```ini
[pytest]
pythonpath = .
testpaths = tests
```

---

## 3) Run the API server

From project root:

```bash
source .venv/bin/activate
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

Expected logs include:
- `Uvicorn running on http://0.0.0.0:8000`
- `Application startup complete`

> If you get `ModuleNotFoundError: No module named 'app'`, you are likely not in project root.

## 6) Submit tasks

Example uploads (mixed tiers):

```bash
curl -s -X POST http://localhost:8000/upload -F "file=@/etc/hosts" -F "user_id=u_free" -F "tier=free"
curl -s -X POST http://localhost:8000/upload -F "file=@/etc/hosts" -F "user_id=u_t3"   -F "tier=tier3"
curl -s -X POST http://localhost:8000/upload -F "file=@/etc/hosts" -F "user_id=u_t1"   -F "tier=tier1"
curl -s -X POST http://localhost:8000/upload -F "file=@/etc/hosts" -F "user_id=u_t2"   -F "tier=tier2"
```

Each response returns:
- `task_id`
- `status` (`queued`)
- `priority`

Save task IDs for later checks.

## 7) Check queue status

## 8) Process tasks (manual worker tick)

Each tick processes **one** task:

```bash
curl -s -X POST http://localhost:8000/worker/tick | python -m json.tool
```

Run repeatedly until queue is empty.

Expected processing order:
1. tier1
2. tier2
3. tier3
4. free

(with FIFO inside same tier)

## Common issues & fixes

### A) `ModuleNotFoundError: No module named 'app'`
- Run uvicorn from project root:
  ```bash
  cd ~/task-queue/restate-taskqueue-demo
  python -m uvicorn app.main:app --reload
  ```

### B) `command not found` for `pythonpath = .` / `testpaths = tests`
- Those lines belong in `pytest.ini`, not terminal.

### C) No output files
- Tasks are still queued.
- Call:
  ```bash
  curl -X POST http://localhost:8000/worker/tick
  ```
